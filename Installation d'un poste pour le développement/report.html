Forme et Lieu de Sauvegarde

Les fichiers de la machine virtuelle sont sauvegardés sur un disque dur local, avec les éléments suivants :

  • Fichier disque virtuel : .vdi, .vmdk, etc.
  • Fichiers de configuration :.vbox, .vmx.
  • Snapshots : Sauvegardes d'états intermédiaires.

Le choix d'un SSD améliore les performances, tandis qu'un disque externe peut être utilisé pour des sauvegardes supplémentaires.

Choix Techniques

Nous avons choisi Ubuntu 22.04 pour sa légèreté, sa gratuité et sa compatibilité avec notre environnement de développement.

  • Processeur : 2 cœurs pour un bon compromis entre performance et disponibilité des ressources.
  • Mémoire vive : 4 Go pour garantir une exécution fluide.
  • Stockage : Disque dynamique de 20 Go pour économiser de l'espace disque.
  • Réseau : Mode NAT pour un accès facile à Internet.

Le mode ACCÈS PAR PONT a été (ensuit) choisi pour permettre à la machine virtuelle de se connecter à Internet sans configuration réseau complexe et afin de poster le serveur hébergeant le site en ligne.

Création des Utilisateurs

Nous avons rencontré un problème à cause de la norme de Linux concernant la nomination des utilisateurs donc il fallait utiliser l'option --allow-bad-names :

  • Création d’un compte administrateur :
    • sudo adduser --allow-bad-names admin_Gr101_BinomeG4
    • sudo usermod -aG sudo admin_Gr101_BinomeG4
  • Création de deux utilisateurs BinomeYY_1 et BinomeYY_2 :
    • sudo adduser BinomeG4_1
    • sudo adduser BinomeG4_2
  • Création d’un groupe BinomeYY:
    1. sudo groupadd BinomeG4
    2. sudo usermod -aG BinomeG4 BinomeG4_1
    3. sudo usermod -aG BinomeG4 BinomeG4_2
  • Personnalisation d’une invite de commande :
    1. taper : nano ~/.bashrc
    2. Localiser la section PS1 et modifier la variable : PS1='(\[\033[01;31m\]\!\[\033[00m\]) \[\033[01;32m\]\u@:\[\033[01;34m\]\w\[\033[00m\]\$

      Cette modification permet d'afficher : le numéro l'historique de la commande entre parenthèses, en rouge puis l'utilisateur suivi de '@' et de ':' , en vert ensuit, le répertoire courant en bleu , enfin affiche '#' si l'utilisateur est root et '$' s'il est un utilisateur normal.

    3. Recharger le fichier .bashrc pour que les modifications soient prises en compte : source ~/.bashrc

Installation et configuration d’un service web

Nous avons décidé de choisir et d'installer le serveur Apache 2.

  1. Installation d'Apache : sudo apt install apache2
  2. Configuration du site web local :
    • Creation d'un repertoire contenat le site web : mkdir monsite.fr
    • Ajouter les fichiers nécessaires comme index.html .css etc.
    • Creation d'un lien symbolique pour le repertoire du site web : ln — s /home/yaz/monsite.fr /unDossier/monsite.fr
    • Déplacer le lien de site dans le répertoire dédié aux sites web (srv) : mv /unDossier/monsite.fr /srv/monsite.fr
    • Donner les permissions nécessaires (droit d'éxecution) pour le repertoire de travail : sudo chmod 751 /home/yaz
  3. Mettre en ligne le site web local :
    • Modifier le fichier '000-default.conf' : dans la section ServerAdmin webmaster@localhost, modifier DocumentRoot /var/www/html en DocumentRoot /srv/monsite.fr.
    • Modifier le fichier 'apache2.conf' : activer la section <Directory> /srv/> en enlevant les commentaires dessus.
